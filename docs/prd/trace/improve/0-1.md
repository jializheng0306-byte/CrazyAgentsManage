## 二、数据流与联动关系

```
SessionDB (SQLite)
├── sessions 表 ──→ /api/dashboard/sessions (列表)
│                 ──→ /api/dashboard/session/<id> (详情+messages)
│                 ──→ /api/dashboard/stats (聚合统计)
│                 ──→ /api/dashboard/stream (SSE 3秒轮询)
└── messages 表 ──┘
         │
    ┌────┼────┐
    ▼    ▼    ▼
  Trace  Events  Streams
 (Gantt) (列表)  (SSE)
  同一数据  纯文本  独立数据源
```

联动关系:
1. 自动刷新(C-4-d) → 每5秒 → loadLatestSession() → 更新D/G区域
2. 新会话事件(Streams) → 触发 loadLatestSession() → 刷新面板
3. 搜索(F-2) → 纯前端DOM过滤 → 不请求后端
4. 缩放(H-1/2/3) → 纯前端重绘 → 不请求后端
5. 点击span(G-2-h) → console.log（待实现详情弹窗）
6. "查看全部会话"(C-3/B-5/C-4-c) → 跳转 /sessions 页面

## 四、性能问题分析

### 加载慢的根因

1. **API查询无索引优化**: `/api/dashboard/sessions` 和 `/api/dashboard/session/<id>` 对 messages 表做全表扫描
2. **session detail 返回全部messages**: 一个会话可能有数百条消息，一次性全部加载
3. **5秒自动刷新**: 每次刷新都重新请求完整数据
4. **无前端缓存**: 每次tab切换都重新渲染
