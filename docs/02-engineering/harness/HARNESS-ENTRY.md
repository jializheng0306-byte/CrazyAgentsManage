# CrazyAgentsManage Harness Entry

## Purpose

This repository adopts the same high-level separation proven in `FlowMindDeploy`, but with a different orchestration surface:

- `Codex ↔ HermesAgent group collaboration` replaces `dual-cli`
- `OMX` remains the runtime/session substrate
- `harness/` remains the durable repository learning layer

## Layer Model

```text
┌──────────────────────────────────────────────────────────┐
│ User / Group Intent                                     │
└──────────────────────────────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────┐
│ Codex ↔ HermesAgent Collaboration Workflow              │
│ - Codex dev ownership                                   │
│ - HermesAgent operations ownership                      │
│ - @mention handoff packets                              │
│ - acceptance / follow-up / closeout                     │
└──────────────────────────────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────┐
│ OMX Runtime Substrate                                    │
│ - state                                                  │
│ - notepad / memory                                       │
│ - trace                                                  │
│ - local runtime resilience                               │
└──────────────────────────────────────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────────┐
│ Repository-Owned Harness Layer                           │
│ - harness/exec-plans                                     │
│ - harness/trace/failures                                 │
│ - harness/trace/successes                                │
│ - harness/memory/*                                       │
└──────────────────────────────────────────────────────────┘
```

## Canonical Rule

- `.omx/` = runtime-local
- `harness/` + `docs/` = repository facts

Never invert that rule.

## Required Artifacts For A Collaboration Round

Every non-trivial Codex ↔ HermesAgent round should leave behind:

1. one runtime state snapshot under `.omx/`
2. one Hermes handoff packet generated by script
3. one durable repository artifact when the round changes shared project truth
4. one harness success/failure record at closeout
